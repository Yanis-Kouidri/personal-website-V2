## Doctorat

Titre : Mon doctorat
Titre 2 : DMX : Data Mule eXpress
Sous titre : "Dorsales de communication éphémères pour la collecte de données dans les réseaux de capteurs sans fil"

Description :
Ma thèse porte sur l'optimisation de la collecte de données dans l'Internet des Objets (IoT). Au sein du projet DMX, je développe des architectures réseaux éphémères capables de s'adapter aux mouvements de collecteurs mobiles (comme des drones ou des véhicules). En combinant des techniques de calculs de bord (edge computing) et de communications opportunistes, mon objectif est de rendre les réseaux de capteurs plus intelligents, plus efficaces et plus économes en énergie, même dans des environnements complexes.

Mots clés : WSN, UAV, Edge computing, routage opportuniste, DTN, multi-RAT, AoI, IoT, Data mule, NTN, 5G, 6G

Bandeau : Thèse CIFRE menée en collaboration entre l'IRIT et Neusta France à Toulouse.
Date : 2026 - 2029

## Projets

### Personal-website-v1

Mon premier gros projet web, c'est l'ancêtre de ce site. L'objectif était pour moi d'apprendre et de mettre en pratique le développement web full stack, j'ai alors choisi la stack MERN (MongoDB Express React Node.js). J'ai alors conçu du front end en React, le backend en Node.js avec Express, la communication entre les deux avec des API REST, le stockage des informations dans une base de données no-SQL MongoDB. J'ai également ajouté un système d'envoi de fichier avec multer et en implémentant un mécanisme CRUD (Create Read Update Delete). J'ai par ailleurs ajouté un système d'authentification et d'inscription en utilise les JSON web token et argon2id pour le hachage des mots de passe et leur stockage sécurisé en base de données. Pour le stockage du contenu du site, j'ai utilisé le CMS Strapi. Au cours de ce projet, j'ai beaucoup appris, cela dit, j'ai décidé de créer une nouvelle version du site beaucoup plus légère, car une infrastructure aussi lourde pour un simple portfolio est selon moi inadaptée.

### Personal-website-v2

M'étant rendu compte que la V1 de mon site commençait à devenir inutilement complexe, j'ai décidé de repartir sur de nouvelles bases en créant la V2 de ce site web que vous consultez actuellement. Ce site web n'est composé que de HTML 5 et CSS 3 statiques construit à parti du Framework Astro. Cela a pour effet de rendre le site extrêmement léger et rapide. Tout le contenu du site est stocké dans des fichiers Markdown. C'était également l'occasion pour moi de tester et d'apprendre à utiliser le framework Astro, très adapté dans le cas d'un site centré sur le contenu. J'ai de plus migré de Node.js à Bun pour la gestion du runtime JS pendant le developpement.
